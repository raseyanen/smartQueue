# SmartQueue Terminal - ESP32-C3 SuperMicro

Прошивка для терминала выдачи талонов в системе SmartQueue.

## Возможности

- **WiFi подключение** - WPA2-Personal и WPA2-Enterprise
- **Режим AP** - автоматический запуск точки доступа при первой настройке
- **Веб-интерфейс** - конфигурация через браузер с паролем
- **OLED дисплей** - поддержка SSD1306 128x64 (опционально)
- **Bluetooth принтер** - печать чеков на матричных принтерах
- **OTA обновления** - загрузка bin файлов через веб-интерфейс
- **Автогенерация хеша** - на основе MAC адреса устройства
- **Индикация** - светодиодная сигнализация состояний
- **Кнопка** - запрос талона и подтверждение печати

## Аппаратные требования

- ESP32-C3 SuperMicro (или совместимая плата)
- OLED дисплей SSD1306 128x64 (опционально)
- Кнопка тактовая
- Светодиод
- Bluetooth матричный принтер (FunnyPrint, Epson и др.)

## Подключение по умолчанию

| Компонент | Пин ESP32-C3 | Настраивается |
|-----------|--------------|---------------|
| Кнопка    | GPIO 9       | Да            |

## Первоначальная настройка

1. После первой прошивки устройство создаст WiFi точку доступа:
   - **SSID**: `SmartQueue-XX:XX` (последние байты MAC)
   - **Пароль**: `smartqueue`

2. Подключитесь к точке доступа и откройте:
   ```
   http://192.168.4.1
   ```

3. В веб-интерфейсе настройте:
   - WiFi SSID и пароль вашей сети
   - URL сервера (например, `http://192.168.1.100:8000`)
   - Ссылку на очередь (из личного кабинета)
   - Пароль на веб-интерфейс
   - Пины GPIO (если отличаются от默认)
   - Включить/выключить OLED

4. Сохраните настройки - устройство перезагрузится

## Веб-интерфейс

### Страницы

- **/** - главная страница со статусом и конфигурацией
- **/save** - сохранение настроек
- **/update** - OTA обновление прошивки

### Параметры конфигурации

| Параметр | Описание |
|----------|----------|
| WiFi SSID | Название WiFi сети |
| WiFi Password | Пароль WiFi |
| Server URL | Адрес Django сервера |
| Queue Link | Короткая ссылка очереди |
| Web Password | Пароль на веб-интерфейс |
| Button Pin | GPIO кнопки |
| LED Pin | GPIO светодиода |
| OLED SDA | GPIO данных дисплея |
| OLED SCL | GPIO тактовой дисплея |
| Enable OLED | Включить дисплей |

## Работа терминала

### Режимы работы

1. **Ожидание** - зелёный индикатор горит постоянно
2. **Запрос талона** - нажмите кнопку
3. **Печать** - после получения талона печатается чек
4. **Подтверждение** - нажмите кнопку ещё раз для подтверждения
5. **Таймаут** - если не подтвердить за 60 секунд, талон отменяется

### Индикация

| Сигнал | Значение |
|--------|----------|
| 3 коротких | Готов к работе |
| 5 коротких | Талон получен |
| 2 коротких | Подтверждено |
| 3 длинных | Ошибка |
| Горит | WiFi подключён |
| Мигает | Подключение/ошибка |

### Чек

На чеке печатается:
```
SMARTQUEUE
----------------
Queue: [название]
TALON: T42
Date: May 22 2026 14:30:00
Hash: a1b2c3d4e5f6...
----------------
[QR код с ссылкой на проверку]
```

## API взаимодействие

    "terminal_hash": " autogenerated from MAC"

## OTA обновление

1. Скачайте bin файл прошивки
2. Откройте веб-интерфейс терминала
3. Перейдите в раздел "OTA Update"
4. Выберите bin файл
5. Нажмите "Upload & Update"
6. Устройство перезагрузится с новой прошивкой

## Генерация хеша

Хеш терминала генерируется автоматически на основе MAC адреса:

```cpp
macAddress = XX:XX:XX:XX:XX:XX
hash = djb2(MAC) -> 16 символов HEX
```

Хеш сохраняется в LittleFS и используется для всех API запросов.

## Устранение проблем

### Не подключается к WiFi
- Проверьте SSID и пароль
- Убедитесь, что сеть 2.4GHz (ESP32 не поддерживает 5GHz)
- Перезагрузите устройство

### Не печатает
- Проверьте подключение принтера по Bluetooth
- Убедитесь, что принтер включён и в режиме сопряжения
- Проверьте пины UART

### Не работает OLED
- Отключите OLED в настройках
- Проверьте подключение SDA/SCL
- Убедитесь, что адрес дисплея 0x3C
